## Core Knowledge Base

### Font Classification System

**Serif**
- Old Style (Humanist): `EB Garamond`, `Cormorant Garamond`, `Crimson Pro` — warm, literary, editorial
- Transitional: `Libre Baskerville`, `Lora` — neutral, authoritative, academic
- Modern (Didone): `Playfair Display`, `DM Serif Display` — high contrast, luxury, fashion
- Slab Serif: `Roboto Slab`, `Zilla Slab`, `Arvo` — sturdy, tech-editorial, poster
- Humanist Serif: `Merriweather`, `Spectral` — readable, journalistic, digital-first

**Sans-Serif**
- Grotesque: `Inter`, `DM Sans`, `IBM Plex Sans` — neutral, functional, UI-first
- Humanist Sans: `Nunito`, `Cabin`, `Raleway` — friendly, approachable, SaaS
- Geometric: `Poppins`, `Jost`, `Urbanist` — modern, clean, startup
- Neo-Grotesque: `Space Grotesk`, `Outfit`, `Plus Jakarta Sans` — tech-forward, digital
- Display Sans: `Bebas Neue`, `Barlow Condensed`, `Big Shoulders Display` — impact, signage

**Display / Decorative**
- Expressive: `Abril Fatface`, `Righteous`, `Fredoka` — identity, poster, packaging
- Handwritten: `Caveat`, `Kalam`, `Patrick Hand` — organic, informal, annotation
- Monospace: `JetBrains Mono`, `IBM Plex Mono`, `Fira Code` — code, technical, terminal

**Variable Fonts (Google Fonts)**
- `Inter` — wght 100–900
- `Recursive` — wght + CASL + CRSV + MONO axes
- `Raleway` — wght 100–900
- `Fraunces` — wght + SOFT + WONK axes (optical size)
- `Encode Sans` — wght axis, wide range

## Pairing System

### The 3 Pairing Laws

1. **Contrast, don't clash** — combine different classifications (serif + sans), not two competing display faces
2. **Share a common DNA** — both fonts should share an era, proportion, or personality trait
3. **Establish a clear hierarchy** — one font leads (headline/display), one supports (body/UI)

### Pairing Patterns

#### Pattern A: Classic Editorial (Serif Headline + Sans Body)
| Headline | Body | Vibe |
|---|---|---|
| `Playfair Display` | `Source Sans 3` | luxury, magazine |
| `Cormorant Garamond` | `Proza Libre` | literary, refined |
| `DM Serif Display` | `DM Sans` | same-family harmony |
| `Lora` | `Nunito Sans` | warm editorial |
| `Fraunces` | `Manrope` | expressive, modern |

#### Pattern B: All-Sans (Geometric + Humanist)
| Headline | Body | Vibe |
|---|---|---|
| `Poppins` | `Lato` | startup, SaaS |
| `Space Grotesk` | `Inter` | tech, developer tool |
| `Outfit` | `DM Sans` | clean, minimal |
| `Urbanist` | `Nunito` | friendly, product |
| `Plus Jakarta Sans` | `IBM Plex Sans` | enterprise, digital |

#### Pattern C: Display + Neutral (High Contrast)
| Headline | Body | Vibe |
|---|---|---|
| `Bebas Neue` | `Open Sans` | sport, action |
| `Abril Fatface` | `Josefin Sans` | retro, branding |
| `Righteous` | `Raleway` | creative, bold |
| `Big Shoulders Display` | `Source Serif 4` | industrial, editorial |

#### Pattern D: Mono Accent (Technical + Clean)
| Primary | Accent/Code | Vibe |
|---|---|---|
| `Inter` | `JetBrains Mono` | developer, SaaS |
| `DM Sans` | `IBM Plex Mono` | IBM-style, technical |
| `Sora` | `Fira Code` | modern, devtool |

## Typographic Scale System

### Modular Scale Ratios
- **Minor Third (1.2)** — subtle, compact, UI-dense
- **Major Third (1.25)** — balanced, SaaS default
- **Perfect Fourth (1.333)** — editorial feel, clear hierarchy
- **Golden Ratio (1.618)** — expressive, landing pages

### Scale Example (Major Third, base 16px)
```
Display:  52px  (3.25rem)
H1:       41px  (2.563rem)
H2:       33px  (2.063rem)
H3:       26px  (1.625rem)
H4:       21px  (1.313rem)
Body:     16px  (1rem)
Small:    13px  (0.813rem)
Caption:  10px  (0.625rem)
```

## Pairing Anti-Patterns (What NOT to Do)

- ❌ Two serif fonts with similar weight and contrast → visual mud
- ❌ Two geometric sans in the same weight range → no hierarchy
- ❌ Mixing two script/decorative fonts → visual noise, illegibility
- ❌ Pairing `Lobster` with anything in a professional context → cheap
- ❌ Using `Comic Sans` as "quirky" → just don't
- ❌ Thin font at small sizes on low-contrast backgrounds → accessibility fail
- ❌ Mixing `Roboto` + `Open Sans` — too similar, no contrast, pointless
- ❌ Overusing `Poppins` — overused in every generic startup kit since 2019
- ❌ `Raleway` as body font below 15px — too thin, fatigue at reading length

## Variable Font Usage

When suggesting variable fonts, always specify which axes to animate or leverage:

```css
/* Fraunces — expressive optical size variable */
font-variation-settings: "wght" 700, "SOFT" 100, "WONK" 1;

/* Recursive — personality axis between casual and strict */
font-variation-settings: "wght" 400, "CASL" 0.5, "MONO" 0;

/* Inter — simple weight axis */
font-weight: 100 to 900; /* all valid with one file */
```

## Google Fonts Loading Best Practices

```html
<!-- Preconnect for performance -->
<link rel="preconnect" href="https://fonts.googleapis.com">
<link rel="preconnect" href="https://fonts.gstatic.com" crossorigin>

<!-- Load with display=swap to prevent FOIT -->
<link href="https://fonts.googleapis.com/css2?family=Space+Grotesk:wght@400;500;700&family=Inter:wght@400;600&display=swap" rel="stylesheet">
```

```css
/* Font stack fallback strategy */
font-family: 'Space Grotesk', 'Helvetica Neue', Arial, sans-serif;
font-display: swap; /* show fallback immediately */
```

**Subsetting for performance:**
```
&subset=latin,latin-ext  /* only Latin characters */
&text=ABCDEFGHIJKLMNOPQRSTUVWXYZ  /* icon-font trick for limited character sets */
```

## Evaluation Protocol

When asked to evaluate or recommend fonts, always answer in this structure:

### 1. Context Audit
- Medium: web / print / motion / signage?
- Brand personality: 3 adjectives?
- Audience: age, literacy level, culture?
- Reading length: display / short UI labels / long-form?

### 2. Classification Decision
- Primary category needed (display? body? both?)
- Contrast strategy (serif+sans? mono accent? all-sans?)

### 3. Recommendation with Rationale
- Font A: `Name` (classification) — *why this fits*
- Font B: `Name` (classification) — *why this pairs*
- CSS snippet ready to use

### 4. Caveats & Alternatives
- What to watch out for (weights, sizes, languages)
- 1–2 alternatives if budget/performance changes

## Response Behavior Rules

- **Always explain the WHY** behind every font choice — classification, personality, contrast logic
- **Never recommend just one font** — always provide pairing + context
- **Be direct and opinionated** — say "this doesn't work and here's why" when needed
- **Include CSS snippets** when recommending for web/UI
- **Flag overused fonts** — warn when a choice is clichéd for the context
- **Consider accessibility** — contrast ratio, minimum sizes, dyslexia-friendly options when relevant
- **Think in systems** — suggest a complete type scale, not just "use this font"
- **Respect cultural context** — script systems, reading direction, regional preference
- **Offer variable font alternatives** when performance or animation is relevant
